## 2. 产品目标与背景

### 2.1 背景

本项目是 VLA 测试应用池（app-pool-plan.md）中的 APP14，定位为**表单密集型 + 日期/时间密集型**应用。

核心目的：
1. **产品价值：** 提供一个纯前端的在线考试系统，支持出题组卷、定时考试、自动评分、成绩查询
2. **测试价值：** 覆盖 Form、Radio、Checkbox、Input、DateTimePicker、Steps、Progress、Collapse、Breadcrumb 等组件的验收场景
3. **视觉差异化：** 采用 "The Scholar's Notebook" 手绘类纸风格，呈现高端速写本质感的考试界面

### 2.2 产品目标

- 用户能快速创建题目（单选/多选/简答/填空四种题型）
- 用户能从题库选题组成试卷，设定分值与时间
- 考生能在线答题，看到实时进度和倒计时
- 系统自动评判客观题（单选/多选/填空），简答题由出题人手动评分
- 用户能查看成绩列表和统计数据

### 2.3 不做什么（明确排除）

- ❌ 后端服务器 — 纯前端应用，数据存 localStorage
- ❌ 用户注册/登录系统 — 无账户体系，考生输入姓名即可
- ❌ 视频监考/防作弊
- ❌ 文件上传题型（图片/附件）
- ❌ 多语言/国际化
- ❌ 移动端专项适配（响应式基本支持即可）

### 2.4 技术约束

- **纯前端 SPA：** 部署在 GitHub Pages，无后端 API
- **数据持久化：** localStorage（含导入/导出 JSON 功能，防数据丢失）
- **时区：** 统一使用浏览器本地时区（`Intl.DateTimeFormat().resolvedOptions().timeZone`），所有时间显示和比较均基于本地时区，不做 UTC 转换

## 3. 用户角色与使用场景

### 3.1 用户角色

| 角色 | 说明 |
|------|------|
| **出题人（Admin）** | 创建题目、组卷、发布考试、查看成绩、手动评分 |
| **考生（Student）** | 参加考试、在线答题、查看自己的成绩和答案回顾 |

> 角色切换：应用顶部提供角色切换入口（Tab/按钮），无需登录，切换即可。默认进入出题人视角。

### 3.2 使用场景（User Stories）

**US-01 创建题目**
> 作为出题人，我想创建不同类型的题目（单选/多选/填空/简答），以便组成试卷。

**US-02 管理题库**
> 作为出题人，我想查看、编辑、删除已有题目，并按题型筛选，以便维护题库。

**US-03 组卷**
> 作为出题人，我想从题库中选择题目组成试卷，设置每题分值和考试时长，以便发布考试。

**US-04 发布/关闭考试**
> 作为出题人，我想设置考试的开始/结束时间并开启考试，以便考生在规定时间内作答。

**US-05 在线答题**
> 作为考生，我想参加已开启的考试，在线作答并提交，以便完成考试。

**US-06 自动评分**
> 作为出题人，我想系统自动评判客观题，只需手动批改简答题，以便高效出成绩。

**US-07 查看成绩**
> 作为出题人，我想查看所有考生的成绩列表和统计数据（平均分/最高分/通过率），以便了解考试情况。

**US-08 答案回顾**
> 作为考生，我想查看自己的答题详情和正确答案，以便复习。

**US-09 数据导入导出**
> 作为出题人，我想导出/导入题库和考试数据（JSON 格式），以便备份和迁移。

## 4. 功能需求

> 每个功能含验收标准（AC），采用 Given-When-Then 格式。AC 编号规则：`AC-{功能编号}-{序号}`

### 4.1 题库管理（F01）

#### 4.1.1 创建题目

出题人通过表单创建题目，支持四种题型：

| 题型 | 字段 | 说明 |
|------|------|------|
| 单选题 | 题干 + 选项（2-6个）+ 正确答案（1个） | Radio 选择 |
| 多选题 | 题干 + 选项（2-8个）+ 正确答案（≥2个） | Checkbox 选择 |
| 填空题 | 题干（含 `____` 占位）+ 参考答案 | Input 填写，精确匹配 |
| 简答题 | 题干 + 参考答案（选填） | Textarea 填写，人工评分 |

**表单字段：**
- 题型选择（Select，必填）
- 题干（Input/Textarea，必填，最长 500 字）
- 选项列表（仅单选/多选，可动态增删，每项最长 200 字）
- 正确答案（单选：Radio 选一个；多选：Checkbox 选多个；填空：Input 填写）
- 分值建议（Input Number，选填，默认 0，范围 0-100）
- 题目标签（Tag，选填，自由输入，如"数学"、"编程"）

**AC-F01-01：** 创建单选题
- Given 出题人在创建题目页
- When 选择题型"单选"，填写题干"1+1=?"，添加选项 A/B/C/D，选中 B 为正确答案，点击保存
- Then 题目保存成功，出现在题库列表中，题型标签显示"单选"

**AC-F01-02：** 创建多选题
- Given 出题人在创建题目页
- When 选择题型"多选"，填写题干，添加 4 个选项，选中 2 个正确答案，点击保存
- Then 题目保存成功，正确答案记录为所选的 2 项

**AC-F01-03：** 创建填空题
- Given 出题人在创建题目页
- When 选择题型"填空"，题干中包含 `____` 占位符，填写参考答案，点击保存
- Then 题目保存成功，参考答案用于自动评分精确匹配（忽略首尾空格）

**AC-F01-04：** 创建简答题
- Given 出题人在创建题目页
- When 选择题型"简答"，填写题干，参考答案留空，点击保存
- Then 题目保存成功，标记为需人工评分

**AC-F01-05：** 表单校验
- Given 出题人在创建题目页
- When 题干为空，或单选/多选题选项少于 2 个，或未指定正确答案
- Then 表单显示对应错误提示（Alert/Message），不允许保存

#### 4.1.2 题库列表与管理

- **列表展示：** Table 显示所有题目，列：序号、题干（截断 50 字）、题型（Tag）、标签、分值建议、创建时间
- **筛选：** 按题型 Tab 筛选（全部/单选/多选/填空/简答）
- **搜索：** 按题干关键字搜索（Input）
- **分页：** 每页 10 条，Pagination 翻页
- **操作：** 编辑（跳转编辑表单）、删除（Popconfirm 确认）
- **空状态：** 无题目时显示 Empty 组件 + 引导创建

**AC-F01-06：** 题型筛选
- Given 题库中有多种题型的题目
- When 出题人点击"多选"Tab
- Then 列表只显示多选题，其他题型隐藏

**AC-F01-07：** 删除题目
- Given 题库中有题目 X
- When 出题人点击删除按钮，Popconfirm 弹出"确认删除？"，点击确认
- Then 题目 X 从列表消失，Message 提示"删除成功"

**AC-F01-08：** 空状态
- Given 题库为空（或筛选后无匹配结果）
- When 页面渲染
- Then 显示 Empty 组件，文案"暂无题目，点击创建第一道题"

### 4.2 组卷（F02）

#### 4.2.1 创建试卷

出题人创建试卷，从题库选题，设置每题分值和考试参数。

**试卷信息表单：**
- 试卷名称（Input，必填，最长 100 字）
- 考试时长（Input Number，必填，单位：分钟，范围 1-480）
- 及格分数线（Input Number，必填，范围 0-总分）
- 考试说明（Input/Textarea，选填，最长 300 字）

**选题方式：**
- 从题库列表勾选（Checkbox）要添加的题目
- 已选题目列表显示在右侧/下方，可调整顺序（上移/下移按钮）和单独设置分值（Input Number）
- 已选题目支持移除

**AC-F02-01：** 创建试卷基本信息
- Given 出题人在创建试卷页
- When 填写试卷名称"期中考试"、时长 90 分钟、及格分 60
- Then 基本信息保存成功

**AC-F02-02：** 从题库选题
- Given 题库中有 20 道题目
- When 出题人勾选 10 道，点击"添加到试卷"
- Then 已选列表显示 10 道题，各题默认分值为题目的分值建议（若为 0 则默认 10 分）

**AC-F02-03：** 调整题目顺序
- Given 已选列表中有题目 A（第 1 题）和 B（第 2 题）
- When 点击 B 的"上移"按钮
- Then B 变为第 1 题，A 变为第 2 题

**AC-F02-04：** 设置题目分值
- Given 已选列表中有题目 A，当前分值 10
- When 将分值修改为 15
- Then 题目 A 在试卷中的分值更新为 15，试卷总分自动重新计算

**AC-F02-05：** 试卷总分自动计算
- Given 已选 3 道题，分值分别为 10、20、30
- When 页面渲染
- Then 显示"总分：60 分"

**AC-F02-06：** 试卷校验
- Given 出题人在创建试卷页
- When 未选任何题目就点击保存
- Then 提示"请至少选择一道题目"

#### 4.2.2 试卷列表与管理

- **列表展示：** Table 显示所有试卷，列：名称、题目数、总分、考试时长、状态（Tag：草稿/已发布/已关闭）、创建时间
- **操作：** 编辑（仅草稿状态）、发布、关闭、删除（Popconfirm）、查看详情
- **分页：** 每页 10 条

**AC-F02-07：** 试卷状态流转
- Given 试卷 X 状态为"草稿"
- When 出题人点击"发布"
- Then 状态变为"已发布"，考生可见此试卷；编辑按钮禁用

**AC-F02-08：** 关闭试卷
- Given 试卷 X 状态为"已发布"
- When 出题人点击"关闭"
- Then 状态变为"已关闭"，考生无法再作答

### 4.3 考试管理（F03）

#### 4.3.1 考试设置

试卷发布时，出题人可设置考试时间窗口：

- **考试开始时间（DateTimePicker，选填）：** 不设置则发布后立即可考
- **考试结束时间（DateTimePicker，选填）：** 不设置则不限截止时间
- **开启/关闭考试（Switch）：** 手动控制考试可用性

**时区规则（锁死）：**
- 所有时间均使用浏览器本地时区
- DateTimePicker 选择的值直接作为本地时间存储和比较
- 显示时间格式：`YYYY-MM-DD HH:mm`
- 不做任何 UTC 转换

**AC-F03-01：** 设置考试时间窗口
- Given 试卷处于草稿状态
- When 设置开始时间为 2026-05-01 09:00，结束时间为 2026-05-01 11:00
- Then 发布后，考生只能在此时间窗口内进入考试

**AC-F03-02：** 开关控制
- Given 试卷已发布，Switch 为"开启"
- When 出题人关闭 Switch
- Then 考试立即不可用，考生端显示"考试已暂停"

**AC-F03-03：** 时间窗口外拒绝进入
- Given 考试时间窗口为 09:00-11:00，当前时间 08:30
- When 考生尝试进入考试
- Then 显示 Alert "考试尚未开始，开始时间：2026-05-01 09:00"

### 4.4 在线答题（F04）

#### 4.4.1 考试入口

- 考生视角的考试列表：Table 显示已发布且在时间窗口内的考试
- 列：试卷名称、题目数、考试时长、状态（可参加/未开始/已结束/已交卷）
- 点击"开始考试"进入答题页

**AC-F04-01：** 考生进入考试
- Given 考生视角，有一场可参加的考试
- When 输入姓名（Input，必填），点击"开始考试"
- Then 进入答题页，倒计时开始

#### 4.4.2 答题页面

**布局：**
- **顶部：** 试卷名称 + 倒计时（Progress 环形或文字）+ 答题进度（Progress 条形，如 "5/20"）
- **导航面包屑：** 考试列表 > 试卷名称 > 答题（Breadcrumb）
- **左侧/顶部：** 题目导航卡片（网格按钮，点击跳转到对应题目，已答标绿、未答标灰、标记标黄）
- **中间主体：** 当前题目卡片（Collapse 展开所有题目或逐题模式）
- **底部：** 上一题 / 下一题 / 标记 / 提交试卷

**答题交互：**

| 题型 | 组件 | 交互 |
|------|------|------|
| 单选 | Radio Group | 点选一个选项 |
| 多选 | Checkbox Group | 勾选多个选项 |
| 填空 | Input | 输入答案文本 |
| 简答 | Textarea | 输入长文本 |

**AC-F04-02：** 答题 — 单选题
- Given 答题页显示一道单选题，4 个选项
- When 考生点选选项 B
- Then 选项 B 被选中（手绘圆圈标记），题目导航中该题标为"已答"

**AC-F04-03：** 答题 — 多选题
- Given 答题页显示一道多选题
- When 考生勾选选项 A 和 C
- Then A 和 C 被选中（手绘打叉标记），可取消勾选

**AC-F04-04：** 题目导航
- Given 试卷有 20 道题，考生已答 8 道，标记 2 道
- When 查看导航面板
- Then 已答题号绿色，标记题号黄色，未答题号灰色

**AC-F04-05：** 自动保存
- Given 考生正在答题
- When 每次选择/输入答案后
- Then 答案自动保存到 localStorage，页面底部显示 Message "已自动保存"（防抖 3 秒）

#### 4.4.3 倒计时与提交

- **倒计时：** 从考试时长开始倒计，最后 5 分钟 Alert 警告（"距离考试结束还有 5 分钟！"）
- **手动提交：** 点击"提交试卷"→ 弹窗确认（Dialog）显示已答/未答统计 → 确认后提交
- **自动提交：** 倒计时结束自动提交，不再弹确认框

**AC-F04-06：** 倒计时警告
- Given 考试剩余时间从 5:00 以上变为 5:00 以下
- When 倒计时达到 5 分钟
- Then 页面顶部出现 Alert 警告"距离考试结束还有 5 分钟！"，字体变红

**AC-F04-07：** 手动提交
- Given 考生已答完所有题
- When 点击"提交试卷"
- Then Dialog 弹出：已答 20/20，确认提交？→ 点击确认 → 跳转到考试结果页

**AC-F04-08：** 自动提交
- Given 倒计时到 0
- When 时间耗尽
- Then 自动提交当前已作答内容，跳转到考试结果页，Message 提示"考试时间已到，已自动提交"

**AC-F04-09：** 未答题提醒
- Given 考生答了 15/20 题
- When 点击"提交试卷"
- Then Dialog 显示"您还有 5 题未作答，确认提交？"

### 4.5 自动评分（F05）

#### 4.5.1 评分规则

| 题型 | 评分方式 | 规则 |
|------|---------|------|
| 单选 | 自动 | 选项与正确答案完全匹配得满分，否则 0 分 |
| 多选 | 自动 | 全部正确得满分；多选/少选/错选均 0 分 |
| 填空 | 自动 | 去除首尾空格后精确匹配（不区分大小写）得满分，否则 0 分 |
| 简答 | 人工 | 出题人在成绩管理页手动打分（Input Number，范围 0-该题满分） |

**AC-F05-01：** 单选自动评分
- Given 考生提交试卷，第 1 题（单选，10 分）正确答案 B，考生选 B
- When 系统评分
- Then 该题得 10 分

**AC-F05-02：** 多选评分 — 全对得分
- Given 第 2 题（多选，10 分）正确答案 A/C，考生选 A/C
- When 系统评分
- Then 该题得 10 分

**AC-F05-03：** 多选评分 — 少选/多选不得分
- Given 正确答案 A/C，考生选 A 或 A/B/C
- When 系统评分
- Then 该题得 0 分

**AC-F05-04：** 填空评分 — 忽略大小写和首尾空格
- Given 正确答案 "Hello World"，考生输入 " hello world "
- When 系统评分
- Then 该题得满分

**AC-F05-05：** 简答题手动评分
- Given 考生提交含简答题的试卷
- When 出题人在成绩页查看该考生详情
- Then 简答题显示考生答案 + 参考答案，提供 Input Number 输入分值，点保存后更新总分

#### 4.5.2 考试结果页

提交后立即展示（Steps 流程：答题→提交→评分→查看结果）：
- **Result 组件：** 显示"考试已提交"或"评分完成"
- **成绩概览：** 总分 / 满分、是否及格（Tag 绿/红）
- **逐题回顾（Collapse 折叠展开）：**
  - 题目 + 考生答案 + 正确答案
  - 正确：蓝铅笔圆圈标记 ✓
  - 错误：红墨水删除线标记 ✗
  - 简答题：显示"待评分"或已评分数

**AC-F05-06：** 考试结果页展示
- Given 考生提交试卷，含 15 道客观题 + 5 道简答题
- When 跳转到结果页
- Then 客观题已评分，显示得分；简答题标记"待评分"；总分为客观题得分（简答部分待评后更新）

### 4.6 成绩管理（F06）

#### 4.6.1 成绩列表

出题人视角：
- **Table 列：** 考试名称、考生姓名、总分/满分、是否及格（Tag）、提交时间、评分状态（全部已评/部分待评）
- **筛选：** 按考试名称（Select）、按评分状态（Tabs：全部/已评/待评）
- **分页：** Pagination，每页 10 条
- **操作：** 查看详情（跳转逐题批改页）、删除成绩记录

#### 4.6.2 统计看板

每场考试的统计信息：
- **Statistic 组件：** 平均分、最高分、最低分、通过率
- **成绩分布：** 用 Progress 条形展示各分段人数占比（0-59 / 60-79 / 80-100）

**AC-F06-01：** 成绩列表
- Given 考试 A 有 5 名考生提交
- When 出题人查看成绩列表
- Then 显示 5 条记录，含分数和及格状态

**AC-F06-02：** 统计数据
- Given 考试 A 有成绩：80, 90, 60, 70, 50
- When 查看统计看板
- Then 平均分 70、最高分 90、最低分 50、通过率 80%（及格线 60，4/5 通过）

**AC-F06-03：** 批改简答题
- Given 考生 B 有 2 道简答题待评分
- When 出题人查看详情，为每道题输入分数，点击保存
- Then 总分更新，评分状态变为"已完成"

### 4.7 数据导入导出（F07）

- **导出：** Button 一键导出全部数据（题库 + 试卷 + 成绩）为 JSON 文件下载
- **导入：** Button 选择 JSON 文件导入，覆盖或合并（Dialog 确认导入模式）

**AC-F07-01：** 导出数据
- Given 系统中有题库和考试数据

**AC-F07-02：** 导入数据

**AC-F07-03：** 导入校验
- Given 导入的文件格式不合法

- Then Alert 提示"文件格式错误，请使用本系统导出的 JSON 文件"

## 5. 业务规则与边界条件

### 5.1 数据约束

| 字段 | 约束 |
|------|------|
| 题干 | 必填，1-500 字 |
| 选项文本 | 必填，1-200 字 |
| 单选题选项数 | 2-6 个 |
| 多选题选项数 | 2-8 个 |
| 多选题正确答案 | ≥2 个，≤ 选项总数 |
| 试卷名称 | 必填，1-100 字 |
| 考试时长 | 1-480 分钟（整数） |
| 题目分值 | 0-100 分（整数） |
| 及格分数线 | 0 至试卷总分（整数） |
| 考生姓名 | 必填，1-50 字 |
| 手动评分分值 | 0 至该题满分（整数） |

### 5.2 状态流转

**试卷状态：**
```
草稿 → 已发布 → 已关闭
         ↑         |
         └─────────┘ (可重新发布)
```
- 草稿：可编辑、可删除
- 已发布：不可编辑，可关闭
- 已关闭：不可编辑，可重新发布（考生已有成绩保留）

**考生答卷状态：**
```
进行中 → 已提交 → 已评分（含简答题时）
```
- 进行中：答题中，可实时保存
- 已提交：客观题已自动评分
- 已评分：所有题目（含简答）评分完成

### 5.3 同名考生处理

- 纯前端无账户系统，考生以姓名标识
- 同一场考试中，若考生输入相同姓名：允许重复提交，按提交时间区分（列表显示提交时间）
- 不阻止重复提交，但成绩列表中每次提交是独立记录

### 5.4 考试时间窗口规则

- 开始时间和结束时间均选填
- 两者都不设：发布后立即可考，无截止
- 只设开始时间：到时间后可考，无截止
- 只设结束时间：立即可考，到时间后截止
- 两者都设：在窗口期内可考
- 结束时间必须晚于开始时间（前端校验）
- 考试进行中到达结束时间：自动提交

### 5.5 localStorage 限制

- 浏览器 localStorage 通常 5-10MB
- 系统在保存前检测剩余空间，不足时 Alert 提示"存储空间不足，请导出数据后清理"
- 建议上限：题库 1000 题，试卷 50 份，成绩 500 条

## 6. 异常与错误设计

### 6.1 表单错误

| 场景 | 提示方式 | 提示内容 |
|------|---------|---------|
| 必填项为空 | Form 行内红字 | "{字段名}不能为空" |
| 超出字数限制 | Form 行内红字 | "{字段名}不能超过 {N} 字" |
| 数值超范围 | Form 行内红字 | "{字段名}范围为 {min}-{max}" |
| 选项不足 | Form 行内红字 | "至少添加 2 个选项" |
| 未设正确答案 | Form 行内红字 + Alert | "请设置正确答案" |
| 多选正确答案不足 | Form 行内红字 | "多选题至少选择 2 个正确答案" |
| 试卷未选题目 | Alert | "请至少选择一道题目" |

### 6.2 运行时异常

| 场景 | 处理 |
|------|------|
| localStorage 写入失败 | Alert "存储失败，请检查浏览器存储空间" |
| localStorage 数据损坏 | Alert "数据异常，建议从备份导入" + 提供重置按钮 |
| 导入 JSON 格式错误 | Alert "文件格式错误，请使用本系统导出的 JSON 文件" |
| 导入 JSON 版本不兼容 | Alert "数据版本不兼容，当前系统版本 v1.0" |
| 考试时间窗口冲突 | Alert "结束时间必须晚于开始时间" |

### 6.3 边界场景

| 场景 | 行为 |
|------|------|
| 答题中刷新页面 | 从 localStorage 恢复答案和倒计时（记录开始时间，用当前时间减去开始时间计算已过时长） |
| 答题中关闭浏览器 | 同上，下次打开恢复 |
| 倒计时精度 | 使用 `Date.now()` 对比开始时间，避免 `setInterval` 漂移 |
| 大量数据渲染 | 题库/成绩超过 100 条时，Table 启用虚拟滚动或严格分页 |

## 8. 视觉风格与设计规范

### Vibe Concept: "The Scholar's Notebook"

> 一份在高端速写本上完成的实体纸质考试。追求建筑精度级的手绘感，避免幼稚涂鸦。

### 8.1 色板

| Token | 色值 | 用途 |
|-------|------|------|
| Paper Background | `#FCF9F2` | 主背景，温暖米白"奶油纸" |
| Ink Dark | `#2C2C2C` | 主要文字 |
| Ink Medium | `#5A5A5A` | 次要文字 |
| Ink Light | `#9A9A9A` | 占位符、禁用态 |
| Rule Blue | `#B8D4E3` (0.3 opacity) | 考试区域淡蓝网格线 |
| Highlighter Yellow | `#FFFF0088` | 当前活跃题目高亮 |
| Blue Pencil | `#4A7CB5` | 正确答案标记 |
| Red Ink | `#C0392B` | 错误答案删除线 |
| Green Ink | `#27AE60` | 及格/已答标记 |
| Tape Beige | `#F0E6D0` | 按钮/卡片"胶带"底色 |

### 8.2 字体

| 角色 | 字体 | 回退 | 用途 |
|------|------|------|------|
| 印刷体（题目/标题） | `'Playfair Display'` | `'EB Garamond'`, `Georgia`, serif | 题干、标题、选项文字 |
| 手写体（用户输入） | `'Architects Daughter'` | `'Indie Flower'`, cursive | 考生答案、姓名输入、手写标注 |
| 等宽体 | `'Courier New'` | monospace | 代码相关（如有） |

> 所有字体从 Google Fonts 加载。

### 8.3 纸质纹理

- **背景：** `#FCF9F2` + CSS grain overlay（noise）模拟纸张纤维
  ```css
  .paper-bg::after {
    content: '';
    position: fixed;
    inset: 0;
    background-image: url("data:image/svg+xml,..."); /* noise pattern */
    opacity: 0.03;
    pointer-events: none;
  }
  ```
- **考试区域网格：** 淡蓝水平线或数学方格纸纹理，仅在考试区域内显示
- **边框不规则：** 主容器使用 wobbly border-radius：
  ```css
  .main-container {
    border-radius: 255px 15px 225px 15px / 15px 225px 15px 255px;
    border: 2px solid #2C2C2C;
  }
  ```

### 8.4 手绘组件规范

**边框（Rough Stroke）：**
- 所有按钮、输入框、卡片使用 2px 不完全笔直的边框
- 实现方式：CSS `border-radius` 不规则值 + 轻微 `rotate` 或 SVG filter

**单选/复选（Radio/Checkbox）：**
- 未选中：手绘空心圆/方框
- 选中 — Radio：手绘有力的圆圈涂鸦（scribble circle）
- 选中 — Checkbox：手绘 "✗" 叉号
- 不使用标准蓝点/对勾

**题目卡片（Question Cards）：**
- 底部"撕纸"边缘效果（torn paper edge，CSS clip-path 或 SVG mask）
- 当前活跃题目：`#FFFF0088` 荧光笔黄底色

**反馈标记：**
- 正确答案：蓝铅笔圆圈（Blue Pencil `#4A7CB5`）
- 错误答案：红墨水删除线（Red Ink `#C0392B`，`text-decoration: line-through`）

**按钮（Buttons）：**
- 主要按钮：胶带贴纸风格（Tape Beige 底色 + 手绘边框）或墨水填充方块
- 次要按钮：仅手绘边框，无填充

**图标（Icons）：**
- Lucide 风格但带 "Scribble" 手绘质感
- 可通过 CSS filter 或手绘 SVG 变体实现

### 8.5 交互动效

**Hover 抖动：**
```css
.interactive:hover {
  animation: wiggle 0.3s ease-in-out;
}
@keyframes wiggle {
  0%, 100% { transform: rotate(0deg); }
  25% { transform: rotate(-1deg); }
  75% { transform: rotate(1deg); }
}
```

**选中动画：** Radio/Checkbox 选中时，手绘标记以"画出"的动画出现（stroke-dasharray 动画）

**页面过渡：** 题目切换时轻微 fade + 微移动（translateY 5px）

### 8.6 响应式要求

- **最小支持宽度：** 768px（平板横屏及以上）
- 窄屏时题目导航面板从侧边转为顶部折叠
- 手机端不做专项优化，保证不破版即可

## 11. 风险与决策记录

### 11.1 已识别风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| localStorage 5MB 限制 | 大量题目/成绩可能撑满 | 提供导出功能 + 容量预警 |
| 浏览器时区差异 | 不同时区用户看到不同考试时间 | 锁死使用浏览器本地时区，不做 UTC 转换（PRD §2.4 & §5.4） |
| 倒计时刷新丢失 | 考生刷新页面倒计时重置 | 记录考试开始时间到 localStorage，刷新后重新计算剩余时间 |
| 纯前端无权限控制 | 考生可访问出题人页面 | 仅做 UI 层隔离（角色切换），不做安全防护（产品定位非高安全场景） |
| 手绘风格增加开发复杂度 | CSS 实现手绘效果耗时 | 优先实现功能，视觉风格作为独立 CSS 层叠加 |

### 11.2 决策记录

| 决策 | 选项 | 选择 | 原因 |
|------|------|------|------|
| 数据存储 | localStorage vs IndexedDB | localStorage | 更简单，5MB 够用，纯前端场景 |
| 多选评分 | 半对给一半分 vs 全对才给分 | 全对才给分 | 规则更简单明确，避免评分争议 |
| 填空匹配 | 精确匹配 vs 模糊匹配 | 精确匹配（忽略大小写和首尾空格） | 模糊匹配太复杂，精确匹配已经足够 |
| 角色切换 | 登录系统 vs UI 切换 | UI 切换 | 纯前端无后端，保持简单 |
| 考试时间 | UTC 统一 vs 本地时区 | 本地时区 | 纯前端无法可靠同步时区，使用本地时间最直观 |

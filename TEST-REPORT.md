## 2. Bug 清单

### BUG-001: 提交对话框已答题数统计为0 (L1.10)

| 项目 | 详情 |
|------|------|
| **用例** | L1.10 手动提交试卷 (`oes-exam-v1.0-L1.10-001`) |
| **严重等级** | **P2 (Medium)** |
| **预期** | 确认对话框显示"已答：8/8 题" |
| **实际** | 确认对话框显示 **"已答：0/8 题"** |
| **复现条件** | Pre-flight 通过 localStorage 注入 8 道题答案后点击提交 |
| **根因分析** | 提交对话框的已答统计仅统计通过 UI 交互产生的答案，不读取 localStorage 中已有的答案数据。注入的答案实际已保存（提交后得分 40/100），但 UI 层计数器未同步 |
| **影响** | 考生断线恢复答题后，提交确认对话框可能显示错误的已答数 |

### BUG-002: 及格考生得分数字显示为红色 (L1.11)

| 项目 | 详情 |
|------|------|
| **用例** | L1.11 单选自动评分 (`oes-exam-v1.0-L1.11-001`) |
| **严重等级** | **P3 (Low)** |
| **预期** | 考生得分 92 分（及格线 60），得分数字应为绿色 |
| **实际** | 得分数字 "92" 显示为 **红色**，"及格"状态通过旁边的绿色按钮体现 |
| **根因分析** | 结果页得分数字颜色未根据及格/不及格状态做条件渲染，统一使用红色（强调色） |
| **影响** | 视觉误导——用户可能误以为不及格 |

### BUG-003: 自动保存无可见提示 (L3.04)

| 项目 | 详情 |
|------|------|
| **用例** | L3.04 自动保存 (`oes-exam-v1.0-L3.04-001`) |
| **严重等级** | **P3 (Low)** |
| **预期** | 选择答案后等待 3-4 秒，页面底部出现"已自动保存"提示 |
| **实际** | 答案已保存（导航面板更新为绿色，进度 1/8），但 **页面底部未出现可见的自动保存提示** |
| **根因分析** | 自动保存 Toast 可能显示时间过短（<1 秒）导致截图捕捉不到，或提示功能未实现 |
| **影响** | 用户无法确认答案是否已保存，缺乏安全感 |
